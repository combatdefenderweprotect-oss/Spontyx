🔴 Missing Bridge: Venue ↔ Player
venue-live-floor.html
✕ no link ✕
live.html
The venue pushes questions from venue-live-floor but there's no connection to the player's live.html. These two pages represent the same match from different perspectives but have zero shared data or linking.
Critical Gaps
🔴
No venue ↔ player bridge Architecture
The venue system (9 pages) and player system (16 pages) are completely isolated. A venue pushes questions on venue-live-floor.html, but there's no mechanism for those to appear on the player's live.html. This is the core gameplay loop and it's disconnected.
🔴
No authentication flow Auth
index.html has signup/login modals but they're UI stubs. No auth state exists — every page shows hardcoded user data. No way to distinguish player vs venue owner roles.
🔴
Zero data persistence (except Table Map) Data
26 of 27 pages have fully hardcoded data. Creating a league, joining a match, answering a question — none of it persists. Only venue-table-map.html uses localStorage.
🔴
No onboarding flow UX
After signup, users land on dashboard.html with no guidance. No interest selection, no tutorial, no first-league prompt. Same gap on venue side — no setup wizard.
UI/UX Issues
🟠
Inconsistent sidebar across pages Nav
league.html is missing "Live" and "Game Modes" sections. battle-royale.html only links back to dashboard. br-leaderboard.html includes itself in sidebar. Navigation should be identical across all player pages.
🟠
Broken filter/tab interactions on 7+ pages Interactivity
discover, matches, upcoming, leaderboard, notifications, venue-schedule, venue-tonights-events — all have filter chips/tabs that visually toggle but don't actually filter content.
🟠
create-league.html wizard doesn't carry state Forms
Step navigation works visually, but selections from Step 1 don't populate the Review in Step 4. Back button may not preserve choices.
🟠
Orphaned pages: create-league.html, live.html Nav
These pages exist but aren't in the main sidebar. Only reachable via specific action buttons or direct URL — could confuse users.
Suggestions
💡
Add shared CSS/JS foundation
All 27 pages duplicate the same CSS variables, sidebar HTML, and utility functions. Extract into shared styles.css, sidebar.html partial, and utils.js. This will make future changes much easier.
💡
Make filters functional with JS array filtering
Even without a backend, you can make filters work by storing mock data in JS arrays and rendering filtered results on click. This makes the prototype feel much more real.
🟣
Build a QR code / join flow for venue → player
Venues need a way to get players into their event. A QR code on venue-live-floor.html that links to a join page would bridge the two systems. This is a key missing piece.
🟣
Add venue-questions.html for question bank
Venues need to manage their custom questions somewhere. Currently venue-create-event only has "auto-push AI questions" toggle but no way to create or browse questions.
